# Greeting App (Django)

Небольшое учебное Django‑приложение:
- главная страница с формой для ввода имени;
- сохранение приветствий в БД;
- страница истории (последние записи);
- небольшой JSON API для дальнейшего расширения.

## Быстрый старт

```bash
pip install -r requirements.txt
python manage.py migrate
python manage.py runserver
```

Откройте в браузере: http://127.0.0.1:8000/

## Тестирование

Базовые тесты находятся в `greeting/tests.py`.

```bash
python manage.py test
```

## Структура проекта

- `greeting/models.py` — модель `Greeting` (имя + дата/время создания)
- `greeting/forms.py` — валидация и нормализация имени
- `greeting/views.py` — страницы и API (PRG + сообщения пользователю)
- `greeting/templates/` — шаблоны (Bootstrap + сообщения/ошибки)
- `greeting/static/` — небольшой кастомный CSS

## UX/визуальная обратная связь

- Используется **Post/Redirect/Get**, чтобы избежать повторной отправки формы при обновлении страницы.
- Сообщения пользователю показываются через **Django messages** (успех/ошибки).
- На кнопке отправки показывается **спиннер** и она блокируется, пока форма отправляется.
- На главной странице отображаются последние приветствия.

## Мини‑API

- `GET /api/greetings/?limit=20` — возвращает JSON со списком последних записей.

Это «заготовка» для будущего: пагинация, фильтры, аутентификация и т.д.

## Production roadmap (план улучшений для реального продакшена)

**СУБД/производительность**
- перейти с SQLite на PostgreSQL (или другую серверную СУБД);
- вынести настройки подключения в переменные окружения;
- добавить индексы/оптимизацию запросов при росте объёма данных.

**Защита данных и безопасность**
- хранить секреты только в переменных окружения (SECRET_KEY, DB creds);
- `DEBUG=False`, заполнить `ALLOWED_HOSTS`;
- включить HTTPS: `SECURE_SSL_REDIRECT`, secure cookies (`SESSION_COOKIE_SECURE`, `CSRF_COOKIE_SECURE`);
- ограничить сбор персональных данных (минимизация), добавить политику хранения/удаления;
- настроить резервное копирование, мониторинг и обновления зависимостей.

**Расширение функциональности**
- добавить дополнительные страницы (статистика, профили, админ‑панель);
- расширять API (пагинация, версии, auth/JWT);
- внедрить более сложные модели данных и бизнес‑логику;
- добавить CI (запуск `python manage.py test` на PR/commit).
